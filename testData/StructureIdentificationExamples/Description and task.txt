These are estimates for which features should be identified on each SAXS data

Values in the table below are estimated ranges from graphs and are not exact. 

All identification/ordering starts from high-Q, tables are ordered from high-q end. Unified fit level 1 is the level for smallest features = high Q range of data. Return order from high q end. 



Practical information: 
Qmin and Qmax of most data should be between 0.00001 to 0.6 [1/A].
If data extend beyond 0.6 [1/A], search will be limited to Qmax of 0.6 [1/A] since that is limit of SAS approximations. Higher Q values are diffraction. 
Qmin is limited by instrumentation. Most data will span 0.001 to 0.6 [1/A], USAXS data span 0.00001 to 0.3, when merged with SAXS data higher. 


We want to identify Background range. We assume flat background. Uncertainties at background level may be significant.  
It is possible to have background = 0. 
If data exhibit minimum in intensity in at high Q (typically above 0.1 [1/A]) in that case a small area around the minimum is estimate for background location. . 

Code "PLS" in table is Power law slope range. We want to identify Power law slope range in Q.    

Code "GK" is Guinier knee . We want to identify range in Q. 

Code "GP" is Guinier Plateau. We want to identify locations range in Q.   

It is user or AI agent job to interpret these features further. User or agent can use local fit tools in Unified fit to obtain parameters, these tools require as input only the Q range. However, the design question is if we want to return more information from the search function as it may already exist here - we can return estimated power law slope for PLS and intensity in the middle of the Q range for GK and GP. 

Note, that Power law slopes ranges are the most important part of the information here. With proper segmentation into power law slopes ranges users/agent may be able to interpret the data already.
